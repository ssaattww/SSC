## 指摘事項

- 指摘要約または「指摘なし」:
  1. T-042 の最小スコープは dynamic value-path `GetState` であり、generated nested value path parity まで同時に広げると task 境界を越える。
  2. non-invasive 化の要件は「`GetState` 中に getter を実行しないこと」であり、値評価の前倒し自体を全面禁止するものではない。
  3. 観測可能な差分は getter の副作用・例外の発生タイミングだけなので、compare / node construction 時へ移ることを明示すれば設計意図を狭く保てる。
  4. generated については公開 `ValueState` の意味だけを dynamic と揃え、実装経路の統一は T-042 の対象外とするのが最も狭い解釈である。

## 結果

- 結果:
  - `doc/design/detail/02-PublicApi.md` に T-042 の dynamic value-path `GetState` scope を追加し、getter 非実行保証と timing change、generated 対象外を明記した。
  - `doc/design/detail/06-ExecutionPipeline.md` に、dynamic value-path `GetState` のため comparable non-container member を compare / node construction 時に internal member node として materialize する設計意図を追記した。
  - `doc/design/detail/09-ValueStateBehavior.md` で value-path state を dynamic と generated に分け、dynamic 側だけ non-invasive state lookup と timing change を規定した。
  - 曖昧だった generated parity は「公開 `ValueState` の意味は同じだが、nested value path の non-invasive 化は T-042 対象外」という狭い解釈で固定した。
